
## **[프로젝트 명: StockSync (엑셀 기반 B2B 실시간 재고 관리 시스템)]**

### **1. 프로젝트 개요**
* **목적:** 엑셀 파일 의존도가 높은 기업의 재고 관리 업무를 웹 시스템으로 전환하여 데이터 투명성 확보 및 실시간 재고 파악 지원.
* **핵심 가치:** "기존 업무 방식(엑셀)을 유지하면서도 시스템의 이점(DB 관리)을 누린다."
* **주요 타겟:** 재고 관리를 엑셀로 하고 있어 버전 관리, 히스토리 추적에 어려움을 겪는 중소/중견 기업 (B2B).

### **2. 핵심 프로세스 (Workflow)**
1.  **기초 데이터 설정:** 최초 재고 리스트 엑셀 업로드로 기준 재고 생성.
2.  **변동 내역 발생:** 판매, 입고, 불량, 반품 내역이 담긴 엑셀 파일 업로드.
3.  **데이터 파싱 및 검증:** 시스템이 엑셀 데이터를 읽어 유효성(SKU 존재 여부, 수량 형식 등) 검사.
4.  **트랜잭션 기록:** 검증된 데이터를 '입출고 내역'으로 DB에 저장 (원본 덮어쓰기 금지).
5.  **재고 산출:** `기초 재고 + 입고 총합 - 출고 총합` 로직으로 실시간 재고 계산 및 대시보드 표출.

### **3. 시스템 아키텍처 설계**

#### **A. 데이터베이스 모델링 (ERD 핵심)**
* **Products (상품 마스터)**
    * `product_id` (PK): 고유 식별자
    * `sku` (Unique): 상품 코드 (엑셀 매칭 키)
    * `name`: 상품명
    * `category`: 카테고리
    * `safety_stock`: 안전 재고 수량
    * `current_stock`: 현재 재고 (캐싱용, 트리거로 자동 계산 권장)

* **Inventory_Transactions (재고 변동 내역)**
    * `transaction_id` (PK)
    * `product_id` (FK): 상품 참조
    * `change_type`: 변동 유형 (`IN`: 입고, `OUT`: 출고, `RETURN`: 반품, `DEFECT`: 불량, `ADJUST`: 실사 조정)
    * `quantity`: 수량 (정수)
    * `reference_doc`: 근거 문서명 (예: 20240501_판매내역.xlsx)
    * `created_at`: 등록 일시
    * `created_by`: 등록자

#### **B. 주요 기능 명세 (Functional Requirements)**
1.  **엑셀 파일 처리 (Core)**
    * `.xlsx`, `.csv` 파일 업로드 지원.
    * **컬럼 매핑(Column Mapping):** 사용자 엑셀의 헤더(예: "상품명")와 DB 컬럼(예: "product_name")을 연결하는 UI 제공.
    * **대량 등록 예외 처리:** 100건 중 3건 실패 시, 97건 성공 처리 후 3건의 실패 사유가 담긴 엑셀 파일 다운로드 제공.

2.  **재고 관리 (Inventory)**
    * 실시간 재고 조회 (검색, 필터링).
    * 재고 수동 조정 (엑셀 없이 단건 수정 기능).
    * 안전 재고 미만 알림 (Low Stock Alert).

3.  **이력 관리 (History)**
    * 제품별 입출고 타임라인 조회 (언제, 어떤 파일에 의해 재고가 변했는지 추적).
    * 특정 시점 재고 조회 (Time Travel 조회).

#### **C. 기술 스택 (Tech Stack)**
* **Frontend:** React (데이터 그리드 구현에 용이), Ag-Grid 또는 TanStack Table (엑셀 같은 UI 제공).
* **Backend:** Python (Django 또는 FastAPI) - Pandas 라이브러리를 활용한 강력한 엑셀 데이터 처리.
* **Database:** PostgreSQL (안정적인 트랜잭션 처리), Redis (실시간 재고 캐싱 필요 시).

### **4. 사용자 인터페이스 (UI/UX) 설계 포인트**
* **대시보드:** 전체 재고 현황, 금일 입출고 건수, 재고 부족 알림 위젯.
* **업로드 마법사:** 파일 선택 -> 데이터 미리보기 -> 컬럼 매핑 -> 유효성 검사 결과 -> 최종 승인 단계로 이어지는 UX.
* **그리드 뷰:** 엑셀과 유사하게 필터, 정렬, 셀 복사/붙여넣기가 가능한 웹 그리드 제공.

### **5. 개발 로드맵 (단계별 구현)**
* **Phase 1 (MVP):** 상품 마스터 등록, 엑셀 업로드를 통한 단순 재고 가감(입고/출고), 현재 재고 조회.
* **Phase 2:** 불량/반품 등 다양한 변동 타입 추가, 실패 데이터 다운로드 기능, 컬럼 매핑 저장 기능.
* **Phase 3:** 대시보드 시각화, 기간별 재고 리포트, 사용자 권한 관리.
